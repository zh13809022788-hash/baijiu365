# 白酒独立站执行建议

基于 `Project_Baijiu_Advocate_Plan.pdf` 的落地判断。

## 一句话结论

这个计划的方向有潜力，但原计划太大、太“帝国叙事”，不适合第一阶段直接执行。现在应该先把它收敛成一个可信、可发布、可持续更新的白酒内容站 MVP，而不是一开始就做 10 语种、Headless WordPress、供应链闭环、全球协会和全自动 Agent 流水线。

更靠谱的第一阶段定位：

面向海外读者、海外华人、烈酒爱好者的中国白酒认知与选购指南站。

英文主站优先，其次简体中文。先做权威感和真实判断力，再谈多语言和商业化。

## 原计划里值得保留的部分

1. 选题方向是对的

白酒在英文互联网里的高质量内容供给明显不足。很多内容要么像百科，要么像招商软文，要么只是简单介绍茅台。做一个更清晰、更现代、更实用的 Baijiu guide，有机会。

2. 静态站方向是对的

Cloudflare Pages / Astro / Jamstack 比 Headless WordPress 更适合早期一人公司。成本低、速度快、安全风险小、维护负担轻。

3. “防 AI 降权”的意识是对的

白酒内容很容易写成空泛 AI 文。必须有明确观点、真实场景、可验证信息、产品判断、风险提醒和独家结构。

4. 多语言是未来优势

但不能一开始做 10 个语种。多语言只有在英文核心内容已经被验证后才值得扩张。

5. 评分体系可以保留

100 分品鉴体系有价值，但第一阶段不要装得像权威机构。可以先作为 “BaijiuBox Score” 或 “Tasting Framework” 使用，逐步积累可信度。

## 原计划里需要降级或暂缓的部分

1. 暂缓 Headless WordPress

本地 WordPress + 静态导出 + GitHub Actions + Cloudflare Pages 的链路太重。早期用 Astro 内容集合直接管理 Markdown/MDX 更稳。

建议技术栈：Astro + Markdown/MDX + Cloudflare Pages。

2. 暂缓 10 语种

第一阶段只做英文 `/en/` 和中文 `/zh/`。如果资源有限，甚至先英文单语。

优先级：

- 英文：全球烈酒爱好者、调酒师、搜索流量
- 中文：海外华人、内容母版、后续中文 SEO
- 其他语种：等英文页面有流量后再做

3. 暂缓“全球白酒品鉴家协会”外壳

GBAS 这种设定容易显得过度包装。如果没有真实成员、评审、样品、照片、数据来源，反而会降低信任。

建议先用编辑部/独立指南口径：

- Independent Baijiu Guide
- Baijiu Advocate
- China spirits field notes

等有真实评测、采访、样品和数据后，再升级成协会/标准项目。

4. 暂缓供应链变现闭环

“裁判员与运动员分离”这部分商业想法要谨慎，容易损害内容站信任。第一阶段不应该在页面上暗埋强商业导向。

更好的变现顺序：

- 阶段 1：内容和搜索流量
- 阶段 2：邮件订阅、酒款数据库、下载指南
- 阶段 3：联盟链接、广告、B2B 询盘
- 阶段 4：供应链合作，但必须透明披露

5. 图片 API 自动化可以后置

早期不需要一开始接 Unsplash/Pexels/Pixabay API。先用少量人工筛选、可商用授权图片，压缩为 WebP/AVIF 即可。

## 第一批内容建议

先不要做 30 篇。先做 12 篇高质量核心页。

### 基础流量页

1. What Is Baijiu?
2. What Does Baijiu Taste Like?
3. Types of Baijiu: Strong, Sauce, Light, Rice Aroma
4. How to Drink Baijiu Without Wasting It
5. Baijiu vs Vodka: Why They Are Not the Same

### 转化/实用页

6. Best Baijiu for Beginners

### 差异化内容页

10. Baijiu Flavor Wheel

12. Chinese Food and Baijiu Pairing Guide

## 内容写法标准

每篇文章必须包含：

- 明确判断，不写中性百科腔
- 真实使用场景，比如宴请、自饮、送礼、调酒、海外购买
- 风味描述，不能只说“香气浓郁、回味悠长”
- 风险提醒，比如假酒、开发酒、年份话术、渠道陷阱
- 和海外读者熟悉的酒类做对比，比如 vodka、soju、sake、whisky、mezcal
- FAQ，用自然语言覆盖长尾搜索
- JSON-LD 结构化数据

## 技术执行建议

不要按 PDF 里的 WordPress 路线走。建议直接初始化 Astro：

```bash
cd /d/白酒独立站
npm create astro@latest .
npm install
npm run dev
```

建议模块：

```text
src/content/articles/
src/content/brands/
src/content/glossary/
src/pages/
src/components/
src/layouts/
public/images/
```

第一阶段需要的功能：

- 首页
- 文章页
- 分类页
- 品牌页模板
- glossary 词条页
- SEO title/description/canonical
- JSON-LD
- sitemap
- robots.txt
- responsive layout

暂时不需要：

- 登录
- 评论
- WordPress 后台
- 数据库
- 多 Agent 自动发布
- 10 语种同步
- 复杂图片 API

## 30 天执行路线

### 第 1 周：项目与定位

- 初始化 Astro 项目
- 确定英文站名、域名、Logo 文字版
- 做首页、文章模板、分类模板
- 写 3 篇核心文章：What Is Baijiu、Types of Baijiu、Best Baijiu for Beginners

### 第 2 周：内容骨架

- 补到 8-12 篇文章
- 建立 glossary
- 做品牌页模板：Maotai、Wuliangye、Luzhou Laojiao
- 加 sitemap、robots、基础结构化数据

### 第 3 周：可信度增强

- 给每篇文章加 FAQ
- 增加图片、表格、风味轮、评分框架
- 做 About 页面，解释编辑标准和利益披露
- 检查移动端可读性和速度

### 第 4 周：上线与收录

- 部署 Cloudflare Pages
- 接入 Google Search Console
- 提交 sitemap
- 根据 Search Console 查询词调整标题和内链
- 决定是否扩展中文版本

## 最推荐的下一步

直接把 `D:\白酒独立站` 初始化为 Astro 项目，并先做英文 MVP。不要再继续完善大而全计划书。先让网站跑起来，发布 3-5 篇能被 Google 理解和收录的核心内容。

建议第一版站名可以从这些方向选：

- Baijiu Advocate
- Inside Baijiu
- Baijiu Field Guide
- The Baijiu Guide
- China Spirits Guide

其中 `Inside Baijiu` 和 `Baijiu Field Guide` 更自然，没那么像机构包装；`Baijiu Advocate` 更贴近原 PDF，但稍微有营销感。
